Button

Buttonで操作しやすいUIを作る

Buttonは機能だけでなく、触りやすさと意図の明確さが重要です。ラベル、タップ領域、役割をセットで設計します。

1. 基本文法（最小コード）

Basic
Button("保存") {
    saveProfile()
}

ラベルとアクションをセットで定義します。

2. 完成コード例（初心者向け）

Code Lab

Basic: 主要ボタン

Button {
    submit()
} label: {
    Text("学習を保存")
        .font(.headline)
        .frame(maxWidth: .infinity)
        .padding(.vertical, 12)
}
.buttonStyle(.borderedProminent)
.tint(.teal)

Intermediate: 再利用可能なボタン設計

Button(role: .destructive) {
    removeMemo()
} label: {
    Label("メモを削除", systemImage: "trash")
        .frame(maxWidth: .infinity)
        .padding(.vertical, 12)
}
.buttonStyle(.bordered)
.controlSize(.large)
.contentShape(Rectangle())

3. 発展コードの読み方（中級入口）

ポイント

role指定

破壊的操作には`.destructive`を使い、意味を明示します。

contentShape

見た目だけでなく実際のタップ判定領域を広げられます。

4. つまずきやすいポイント

アンチパターン
  • 重要ボタンと補助ボタンの見た目が同じで優先度が不明。
  • ラベルが短すぎて操作内容が伝わらない。
  • タップ領域が狭く、誤操作が増える。

5. まとめチェックリスト

確認
  • Buttonの基本構文を説明できる。
  • 操作の重要度に応じてスタイルを変えられる。
  • タップ領域を十分に確保できる。
