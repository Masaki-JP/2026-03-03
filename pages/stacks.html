Stacksを使い分けて複合レイアウトを組む

VStackだけでは実務の画面は組みきれません。HStackで横並び、ZStackで重なりを扱い、構造を分離して実装します。

1. 基本文法（最小コード）

HStack

HStack {
    icon
    Text(title)
}

ZStack

ZStack {
    background
    content
}

2. 完成コード例（初心者向け）

Basic: アイコン付き横並びメニュー

HStack(spacing: 10) {
    Image(systemName: "book.fill")
    Text("学習履歴")
    Spacer()
    Image(systemName: "chevron.right")
}
.padding(14)
.background(.white, in: RoundedRectangle(cornerRadius: 12))

Intermediate: ZStackで背景と内容を分離

ZStack(alignment: .bottomLeading) {
    LinearGradient(
        colors: [.teal.opacity(0.7), .blue.opacity(0.5)],
        startPoint: .topLeading,
        endPoint: .bottomTrailing
    )

    VStack(alignment: .leading, spacing: 8) {
        Text(title).font(.headline)
        Text(subtitle).font(.subheadline)
    }
    .padding(16)
}
.frame(height: 180)
.clipShape(RoundedRectangle(cornerRadius: 16))

3. 発展コードの読み方（中級入口）

責務分離

HStackは並び順、ZStackは重なりに責務を分けると見通しが良くなります。

Spacerの利用

HStack内で端揃えを作るときはSpacerを優先し、固定余白を減らします。

4. つまずきやすいポイント

  • ZStackで複数要素を重ねすぎて階層が把握しにくくなる。
  • HStackで固定幅を多用し、画面幅変化に弱くなる。
  • alignment未指定で意図しない配置になる。

5. まとめチェックリスト

  • HStack/ZStackの役割の違いを説明できる。
  • Spacerで可変レイアウトを作れる。
  • 背景とコンテンツを分離した構造で組める。
