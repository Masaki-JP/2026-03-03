Text

Textで情報の階層を作る

Textは「文字を表示するだけ」のビューではありません。 フォント、太さ、行数、色、余白を組み合わせることで、画面の読みやすさと意味の優先順位を決めます。

1. 基本文法（最小コード）

Basic

まずは最小形です。文字列を表示するだけなら1行で書けます。

Text("こんにちは SwiftUI")

ここから `.font` や `.foregroundStyle` を積み上げて、情報の優先度を作ります。

2. 完成コード例（初心者向け）

Code Lab

Basic: 見出しと補足文を並べる

VStack(alignment: .leading, spacing: 8) {
    Text("今日の学習")
        .font(.title3)
        .fontWeight(.bold)

    Text("Textとpaddingの使い分けを練習する")
        .font(.body)
        .foregroundStyle(.secondary)
}
.padding()

Intermediate: 行数制御とアクセシビリティを考慮

VStack(alignment: .leading, spacing: 10) {
    Text(headline)
        .font(.system(.title3, design: .rounded, weight: .bold))
        .lineLimit(2)
        .minimumScaleFactor(0.85)

    Text(summary)
        .font(.subheadline)
        .foregroundStyle(.secondary)
        .lineSpacing(4)
        .accessibilityLabel("概要: \(summary)")
}
.padding(16)
.background(.thinMaterial, in: RoundedRectangle(cornerRadius: 14))

3. 発展コードの読み方（中級入口）

ポイント

lineLimit + minimumScaleFactor

デバイス幅が狭いとき、タイトルが切れすぎないよう調整できます。

lineSpacing

説明文が2行以上になる場合、視線移動が楽になります。

中級者向けの第一歩は「見た目」だけでなく「情報の読み取りやすさ」も設計することです。

4. つまずきやすいポイント

アンチパターン
  • フォントサイズを固定値だけで運用し、Dynamic Typeを考慮しない。
  • 強調色を多用して重要度の差が消える。
  • 長文Textを1ブロックにまとめ、段落分割しない。

5. まとめチェックリスト

確認
  • Textに役割ごとのスタイル（見出し/本文/補足）を適用できる。
  • 行数が増えた場合も読みやすさを保つ設定ができる。
  • BasicとIntermediateの違いを説明できる。
