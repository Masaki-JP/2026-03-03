VStack

VStackで縦方向の情報構造を作る

VStackは多くの画面の骨格です。spacingとalignmentを適切に設定すれば、情報量が増えても読みやすさを維持できます。

1. 基本文法（最小コード）

Basic 
VStack {
    Text("上")
    Text("下")
}

`spacing` を指定しない場合はデフォルト値で並びます。

2. 完成コード例（初心者向け）

Code Lab

Basic: プロフィールカード

VStack(alignment: .leading, spacing: 8) {
    Text("鈴木 一郎").font(.headline)
    Text("iOS Learner").foregroundStyle(.secondary)
    Text("Text / VStack / padding を学習中")
        .font(.footnote)
}
.padding(16)

Intermediate: セクションをネストして整理

VStack(alignment: .leading, spacing: 14) {
    header
    VStack(alignment: .leading, spacing: 8) {
        statusRow
        summary
    }
    footerActions
}
.frame(maxWidth: .infinity, alignment: .leading)
.padding(18)
.background(.white, in: RoundedRectangle(cornerRadius: 16))

3. 発展コードの読み方（中級入口）

ポイント

alignmentを先に決める

先に`leading`/`center`を決めると、子ビューの調整が減ります。

段落ごとにVStackを分割

意味のまとまり単位でネストすると、再利用と保守が楽になります。

4. つまずきやすいポイント

アンチパターン
  • VStackに要素を詰め込みすぎて責務が曖昧になる。
  • spacingを0にしすぎて可読性が落ちる。
  • alignment未指定のまま個別Textで整列を調整してしまう。

5. まとめチェックリスト

確認
  • VStackのalignment/spacingを用途で使い分けられる。
  • 意味単位でネストして構造化できる。
  • frame/paddingとの組み合わせを説明できる。
