paddingで読みやすい余白を設計する

paddingは見た目の微調整ではなく、情報を理解しやすくする設計要素です。方向指定と値の粒度を使い分けましょう。

1. 基本文法（最小コード）

Text("余白のあるラベル")
    .padding()

`padding()` は全方向に同じ余白を適用します。

2. 完成コード例（初心者向け）

Basic: カード内余白を確保

VStack(alignment: .leading, spacing: 6) {
    Text("学習メモ").font(.headline)
    Text("overlayのalignmentを復習")
}
.padding(16)
.background(.white, in: RoundedRectangle(cornerRadius: 12))

Intermediate: 方向ごとに最適化

VStack(alignment: .leading, spacing: 8) {
    header
    body
    footer
}
.padding(.horizontal, 18)
.padding(.top, 14)
.padding(.bottom, 12)
.background(.thinMaterial, in: RoundedRectangle(cornerRadius: 14))

3. 発展コードの読み方（中級入口）

縦横を分ける

`.padding(.horizontal)` と `.padding(.vertical)` を分けると、画面密度の調整がしやすくなります。

背景適用順

paddingの前後でbackgroundを変えると、余白の範囲が変わります。意図を持って順序を決めます。

4. つまずきやすいポイント

  • 全コンポーネントに同じpaddingを機械的に適用してしまう。
  • frameとの組み合わせ順を意識せず、余白が効かない状態になる。
  • 余白不足でタップ領域が狭くなり、操作しにくくなる。

5. まとめチェックリスト

  • 全方向paddingと方向指定paddingを使い分けられる。
  • modifierの順序で結果が変わる理由を説明できる。
  • 読みやすさ・操作しやすさの観点で余白を調整できる。
