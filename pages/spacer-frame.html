Spacer / frame

Spacerとframeで可変幅レイアウトを制御する

画面サイズが変わっても崩れないUIを作るには、固定値よりもSpacerとframeの使い分けが重要です。

1. 基本文法（最小コード）

Basic
HStack {
    Text("左")
    Spacer()
    Text("右")
}

2. 完成コード例（初心者向け）

Code Lab

Basic: タイトルと日時を左右配置

HStack {
    Text("UIレビュー")
        .font(.headline)
    Spacer()
    Text("18:30")
        .font(.subheadline)
        .foregroundStyle(.secondary)
}
.padding(.horizontal, 16)

Intermediate: frameで幅と揃えを統一

VStack(spacing: 12) {
    row(title: "Text", score: 82)
    row(title: "overlay", score: 76)
}
.frame(maxWidth: .infinity, alignment: .leading)
.padding(16)
.background(.white, in: RoundedRectangle(cornerRadius: 16))

3. 発展コードの読み方（中級入口）

ポイント

maxWidth: .infinity

親コンテナに合わせて広げることで、行揃えが安定します。

alignment指定

frameにalignmentを入れると、余白の中での配置を明示できます。

4. つまずきやすいポイント

アンチパターン
  • Spacerで解決できる場面で固定widthを使う。
  • frameを多用しすぎて階層ごとの責務が不明になる。
  • alignmentを親子で重複指定して挙動が読めなくなる。

5. まとめチェックリスト

確認
  • Spacerと固定余白の使い分けができる。
  • frame(maxWidth: .infinity)の効果を説明できる。
  • 画面幅が変わっても崩れにくい行レイアウトを作れる。
